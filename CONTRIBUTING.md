## How Can I Contribute?

### Reporting Bugs
* Check the **Issues** tab to see if the bug has already been reported.
* If not, open a new issue. Clearly describe the problem and include steps to reproduce it.

### Suggesting Enhancements
* Open an issue and describe the feature you'd like to see and why it would be useful.

### Pull Requests (PRs)
1. **Fork** the repository.
2. **Clone** your fork (`git clone https://github.com/YOUR_USERNAME/bolt-ecommerce-flutter-clean.git`).
3. **Create a branch** for your feature or fix (`git checkout -b feature/amazing-feature`).
4. **Commit** your changes (`git commit -m 'feat: add amazing feature'`).
5. **Push** to the branch (`git push origin feature/amazing-feature`).
6. **Open a Pull Request**.

## Coding Standards
* We follow **Clean Architecture**. Please keep Data, Domain, and Presentation layers separate.
* Use **Riverpod** for state management.
* Follow the official [Dart Style Guide](https://dart.dev/guides/language/analysis-options).

## Questions?
If you have any questions, feel free to open an issue or contact the maintainer!
